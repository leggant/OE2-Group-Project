# Overview
## Problem statement
Describe the problem you're trying to solve by doing this work.
1. `Install a configuration management system for your servers. You can choose puppet for that.`
2. `Make sure the configuration management system ensures Vim is installed in all the machines.`
3. `Follow the best practices when you do this task.`
4. `You can consult the lab document`
## Steps to resolution
1- the first step was to install puppetmaster on mgmt machine 
```bash
sudo apt get-install puppetmaster
```
2- the next step is to edit a file in /etc/puppet/puppet.conf and add the server name (group.mgmt.foo.org.nz) then create a file in the following dir /etc/puppet/code/environments/production/manifests/ called site.pp  
then restart puppetmaster 
3 - on the puppet or slave Machine, install puppet 
```bash
sudo apt-get install puppet
```
then run the following command (groupb.mgmt.foo.org.nz) being the puppet master
```bash
sudo puppet agent --server=groupb.mgmt.foo.org.nz --no-daemonize --verbose --onetime
```
this will create a certificate and send it to the puppetmaster to be signed ,
but in our case, we had various issues with the certificate creations as we had several requests yet to be signed, also we had issues with the puppetmaster not starting 
- after installing the puppetmaster , we had other versions installed so that caused some issues starting the puppetmaster, so to resolve it I removed all Packages
```bash
sudo apt remove puppetlabs-release puppet6-release puppet puppet-agent puppet-master
```
then I switched to root with 
```bash
sudo -i 
```
navigated to /var/lib/puppet/ssl and deleted everything the directory, and ran
```bash
sudo puppet cert clean -all
```
 to remove all the pending and faulty certificates.
next, I did the same thing in the puppet machine then navigated back to puppetmaster and ran 
```bash
sudo systemctl restart puppetmaster
```
then 
```jsx
puppet cert clean groupb.db.foo.org.nz
```
after all that we were able to send the certificate to the puppetmaster and sign them 
```bash
sudo puppet cert --sign --all
```
 the next step was to remove vim from db machine and add the following instructions in the site.pp on the masterpuppet machine to ensure that it install vim 
```bash
node 'groupb.db.foo.org.nz' {
    package { 'vim': ensure => present }
}
```
